# Robot Deployment Guide

This directory contains the USB deployment artifact for the RC car robotics platform.
It is generated by running `bundleJetson.sh` or `bundleRPi.sh` from the project root.

---

## Contents

```
usb_package/
  ├── install.sh      ← run this on the robot (platform-specific)
  ├── requirements.txt
  ├── wheels/         ← pre-downloaded .whl files (offline pip install)
  └── src/
      └── main.py     ← robot entry point
```

---

## Step 1 — Format and Label the USB Drive

The udev auto-run rule (Step 4) keys off the USB volume label.
**The drive must be labeled `ROBOTDEPLOY` exactly.**

### On Linux (dev machine):
```bash
# Identify your USB device (e.g. /dev/sdb1) — check `lsblk` before and after plugging in
sudo mkfs.vfat -F 32 -n ROBOTDEPLOY /dev/sdX1    # replace sdX1 with your device
```

### On macOS (dev machine):
```bash
# Identify disk with `diskutil list`, then:
diskutil eraseDisk FAT32 ROBOTDEPLOY /dev/diskN   # replace diskN
```

### On Windows (dev machine):
Right-click the USB drive in File Explorer → Format → File system: FAT32 → Volume label: `ROBOTDEPLOY`

---

## Step 2 — Copy Files to the USB Drive

After formatting, copy the entire `usb_package/` directory contents to the root of the USB drive.
The USB root should look like:

```
ROBOTDEPLOY (USB root)/
├── install.sh
├── requirements.txt
├── wheels/
└── src/
    └── main.py
```

### On Linux/macOS:
```bash
cp -r usb_package/. /media/$USER/ROBOTDEPLOY/
# or wherever your USB is mounted
```

### On Windows:
Copy the contents of `usb_package` into the root of the USB drive via File Explorer.

---

## Step 3 — Run install.sh on the Robot (First-Time Setup)

Plug the USB into the robot. If udev auto-run is not yet configured (it won't be on a fresh microcontroller),
run the install script manually:

### Jetson Nano:
```bash
sudo bash /media/nvidia/ROBOTDEPLOY/install.sh
```

### Raspberry Pi 4:
```bash
sudo bash /media/pi/ROBOTDEPLOY/install.sh
```

This will:
1. Copy `src/` to local storage (`~/robot/src`)
2. Copy `requirements.txt` to `~/robot/`
3. Install all Python dependencies from the local `wheels/` directory (no internet needed)

> **Jetson Nano — OpenCV note:** `opencv-python` is excluded from the wheel bundle.
> On Jetson, OpenCV should be installed via JetPack for hardware acceleration:
> ```bash
> sudo apt-get install python3-opencv
> ```

## Step 4 — systemd Service (Auto-Run on Boot)

Create the service file on the robot. This makes the code start automatically every time the microcontroller powers on.

### Jetson Nano — `/etc/systemd/system/robot.service`:
```ini
[Unit]
Description=DRC Autorun Service
After=local-fs.target
StartLimitIntervalSec=0

[Service]
Type=simple
User=nvidia
WorkingDirectory=/home/nvidia/robot
ExecStart=/usr/bin/python3 /home/nvidia/robot/src/main.py
Restart=on-failure
RestartSec=3
Environment=DISPLAY=:0
SupplementaryGroups=video gpio i2c

[Install]
WantedBy=multi-user.target
```

### Raspberry Pi 4 — `/etc/systemd/system/robot.service`:
```ini
[Unit]
Description=DRC Autorun Service
After=local-fs.target
StartLimitIntervalSec=0

[Service]
Type=simple
User=pi
WorkingDirectory=/home/pi/robot
ExecStart=/usr/bin/python3 /home/pi/robot/src/main.py
Restart=on-failure
RestartSec=3
Environment=DISPLAY=:0
SupplementaryGroups=video gpio i2c

[Install]
WantedBy=multi-user.target
```

### Enable and start the service:
```bash
sudo systemctl daemon-reload
sudo systemctl enable robot.service
sudo systemctl start robot.service
```

### Useful service commands:
```bash
sudo systemctl status robot.service       # check current status
sudo journalctl -u robot.service -f       # live log output
sudo systemctl restart robot.service      # restart after manual changes
sudo systemctl disable robot.service      # stop auto-run on boot
```

## Step 5 — udev Rule (Auto-Run install.sh on USB Plug-In)

Once the robot has been set up once, you can configure it to automatically re-run `install.sh`
whenever the `ROBOTDEPLOY` USB is inserted. This makes re-deploying updated code as simple as
plugging in the USB drive.

Create the following file on the robot:

### `/etc/udev/rules.d/99-robot-usb.rules`
```
ACTION=="add", SUBSYSTEM=="block", ENV{ID_FS_LABEL}=="ROBOTDEPLOY", \
  RUN+="/bin/bash -c 'sleep 2 && mkdir -p /media/robot_usb && mount /dev/%k /media/robot_usb && bash /media/robot_usb/install.sh >> /var/log/robot_install.log 2>&1'"
```

Reload udev rules:
```bash
sudo udevadm control --reload-rules
sudo udevadm trigger
```

After this, plugging in the USB drive will:
1. Mount the drive automatically
2. Run `install.sh` (updating code and dependencies)
3. Log output to `/var/log/robot_install.log`

> **Note:** The `sleep 2` gives the filesystem time to fully mount before the script runs.
> The `install.sh` script does **not** restart the robot service automatically — do this manually
> or add `systemctl restart robot.service` to the end of `install.sh` if desired.

## Re-Deploying Updated Code

Whenever the codebase changes:

1. On the dev machine, re-run the bundle script from the project root:
   ```bash
   ./bundleJetson.sh   # or ./bundleRPi.sh
   ```
2. Copy the new `usb_package/` contents to the USB drive (overwrite existing files)
3. Plug the USB into the robot — `install.sh` runs automatically (if udev is configured)
   or run it manually as shown in Step 3

## Troubleshooting

| Problem | Fix |
|---|---|
| `install.sh: permission denied` | Run `chmod +x install.sh` on the USB, or call with `bash install.sh` |
| Wheels fail to install | Check that the bundle script matched the robot's Python version (`python3 --version`) |
| Service doesn't start | Check logs: `journalctl -u robot.service -n 50` |
| udev rule not triggering | Check label: `lsblk -o NAME,LABEL` — must be exactly `ROBOTDEPLOY` |
| OpenCV import error on Jetson | Install via apt: `sudo apt-get install python3-opencv` |
| Camera not accessible | Add user to `video` group: `sudo usermod -aG video nvidia` (or `pi`) |
